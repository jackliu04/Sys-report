Deployment Manual

Follow the instructions for deployment



ðŸ“Œ Prerequisites



Install the following tools:
1. Conda
2. Docker & Docker Compose
3. Node.js (v18+)
4. npm (bundled with Node.js)

01

Backend Setup

Clone repository and set up environment:


git clone https://github.com/Hao-Hao211/ChatLincs.git
cd ChatLincs
conda env create --platform osx-arm64 -f chatlincs.yml
conda activate chatlincs


Launch Docker services:


cd backend/app
docker compose up -d


Note: The ollama service will attempt to automatically download the llava:7b model on first launch.If the download fails (e.g. due to network restrictions), you can manually pull the model on your host machine:


ollama pull llava:7b


Then ensure the model is saved under the mounted volume path (e.g. ./ollama-models) so it can be used inside the container.

02

Frontend Setup

Install dependencies:


cd ChatLincs/frontend
npm install --legacy-peer-deps

03

Run Backend (Flask)

Run Flask server:


cd backend
python run.py


Backend API at: http://localhost:5000

04

Run Frontend (Next.js)

Launch frontend server:


cd frontend
npm run dev


Frontend at: http://localhost:3000

Privacy of Data

PRIVACY STATEMENT



The NTTDATA Environmental Monitoring Platform respects your privacy and is committed to safeguarding all data handled within our system. This Privacy Statement explains how we manage data within our locally-deployed environmental monitoring application.



1. INFORMATION WE COLLECT: We do not collect any personal data from users. Our system is designed to function without gathering names, contact details, or any other personally identifiable information. All uploaded media (e.g., images, audio, video) is handled anonymously and solely for project-related analysis.



2. DEVICE PERMISSIONS: Our application may require access to local device features (e.g., file uploads for images or audio recordings), but it does not collect or transmit data beyond what is voluntarily submitted for environmental monitoring purposes.



3. DATA SECURITY: All submitted data is stored and processed entirely on local infrastructure. We implement strict access control, and no data is transferred to external servers or third-party platforms. The system operates offline to ensure maximum privacy and security.



4. THIRD-PARTY SERVICES: This platform does not integrate with any external services. No data is shared with third parties. All large language models and databases run locally.



5. CHILDREN'S PRIVACY: Our system is intended for general use and does not knowingly collect data from children. If any identifiable information is inadvertently submitted, please contact us and we will take steps to remove it.



6. CHANGES TO THIS PRIVACY STATEMENT: We may update this Privacy Statement as our system evolves. Any changes will be communicated within the platform. Continued usage implies acceptance of any modifications.
